# **Problem 3 대체 얼마나 산거냐고**
## 배경 지식(Background)
### 중첩 조건문
- **조건문 안에 또 다른 조건문**을 넣는 방식. 큰 분류를 먼저 나누고, 그 안에서 세부 조건을 따질 때 유용하다.
  
  ```
  if (조건):
    if (조건):
      (코드)
    else:
      (코드)
  ```

---

## Question

- 온라인 쇼핑몰의 결제 로직을 설계하시오.
- 사용자의 구매 금액과 회원 여부에 따라 최종 결제 금액과 적립 포인트가 달라진다.
- 조건은 다음과 같다.

 **1. 배송비 계산**
  - 총 구매 금액이 50,000원 이상이면 배송비는 0원
  - 50,000원 미만이면 배송비 3,000원이 추가
    
 **2. 회원 등급별 포인트 적립**
  - 회원(y)인 경우: 구매 금액의 5%를 포인트로 적립한다. (배송비 제외 금액 기준)
  - 비회원(n)인 경우: 구매 금액의 1%를 포인트로 적립한다.
      
 **3. 최종 출력(예시)**
  - `최종 결제 금액(배송비 포함): 100000원`
  - `적립 예정 포인트: 5000점`
